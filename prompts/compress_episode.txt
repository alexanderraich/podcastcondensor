You are an editor condensing a "Lord of Spirits" podcast episode.
Return ONLY valid JSON — no markdown, no extra text.

GOAL: Identify the 1-2 key POINTS this episode makes. For each point,
find the continuous passage(s) across the episode that develop it.
The show is structured into 3 roughly-equal "halves" (thirds), and a
key point is typically introduced, developed, and concluded across
them — collect the relevant stretches from wherever they occur.

Input — full transcript with SRT timestamps:

  Episode: "{title}"
  Episode number: {episode}

  [INDEX] START-END: TEXT
  [   1]   12.0-   16.5: Sample entry text
  [   2]   16.5-   22.0: Next entry text

Output:

{
  "core_idea": "One sentence describing this episode's main argument or topic",
  "segments": [
    {
      "start": 620.0,
      "end": 850.0,
      "reason": "Why this segment is part of a key point — be specific about what is taught here"
    }
  ]
}

RULES:

1. CORE IDEA FIRST. What is this episode actually about? Every segment
   you pick should directly contribute to one of the 1-2 key points.

2. 1-2 KEY POINTS PER EPISODE. Most episodes have exactly one main point.
   A second point is only justified if the episode makes TWO distinct
   substantive arguments. A few continuous segments from across the
   3 halves may be needed to cover each point fully — that's fine.

3. THE 3 HALVES AS A GUIDE (not a quota):
   - First half (~0–33%): usually introduces the topic, defines terms
   - Middle half (~33–66%): develops the argument, exegesis, context
   - Final half (~66–100%): applications, implications, tying together
   A key point may need a segment from each half, or only one. The
   halves help you locate where content lives — don't pad to fill them.

4. DIRECT TIMESTAMPS ONLY. start and end in seconds, matching the SRT
   timestamps in the input. Not cue numbers, not word indices.

5. COMPLETE THOUGHTS ONLY. Each segment must start where the speaker
   begins explaining the idea and end where that idea wraps up. Never
   cut mid-sentence or mid-argument. Widen to the nearest natural
   boundary.

6. NO TRAILING FRAGMENTS. If the segment would start with a trailing
   clause (lowercase first word like "because..." or "and so..."),
   widen to where the full sentence began. If it would end on "and",
   "but", "so", or other hanging words, widen the end.

7. WHAT GETS DROPPED: banter, sponsor breaks, station IDs, prayer
   openings, retweets, reading listener emails, caller pleasantries
   ("great question, thanks for calling"), previews, recaps of previous
   episodes, "we'll get to that later" teases, second+ illustrations
   of the same concept, tangential digressions, outro, credits, and
   anything not building one of the 1-2 key points.

8. WHAT GETS KEPT: The passages where the hosts are actively teaching
   the episode's key points. Exegesis of the primary text. Historical
   or doctrinal claims. A caller Q&A only if the host's substantive
   reply IS a key point (not the pleasantries around it).

9. BE HONEST ABOUT BOUNDARIES. If uncertain where a segment should end,
   err on including MORE rather than cutting too short. Slightly longer
   is better than mid-thought cut.
